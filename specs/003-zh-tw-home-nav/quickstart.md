# Quickstart：zh-TW 語系與首頁返回入口

## 1. 前置準備

1. 在專案根目錄執行 `ruby bin/rails db:prepare`。
2. 啟動應用程式：`ruby bin/rails server`。
3. 如需完整流程資料，可沿用既有測試或 seed/fixture 建立一筆可查看結果的搜尋請求。

## 2. 驗證首頁為繁體中文

1. 開啟首頁 `http://localhost:3000/`。
2. 確認頁面主要標題、欄位標籤、提交按鈕、日期與旅程相關說明皆為繁體中文。
3. 任意輸入不完整條件並送出，確認驗證或錯誤提示為繁體中文。

**預期結果**

- 首頁不再顯示核心英文導覽文案，例如 `Flight Fare Search`、`Search Fares`。
- 中文文案足以讓旅客理解如何開始搜尋。

## 3. 驗證搜尋狀態頁為繁體中文

1. 從首頁送出一筆有效搜尋條件。
2. 進入搜尋狀態頁後，確認頁面標題、摘要欄位名稱、來源狀態區塊與查看結果入口皆為繁體中文。
3. 等待頁面自動輪詢一次，確認請求狀態與來源狀態標示仍維持繁體中文。

**預期結果**

- 成功提示與狀態頁標題為繁體中文。
- 狀態頁沒有殘留核心英文 CTA、標題或狀態標示。

## 4. 驗證搜尋結果頁的返回首頁入口

1. 進入搜尋結果頁。
2. 確認頁面主要操作區可見「回到首頁」入口。
3. 點擊該入口，確認直接返回首頁。

**預期結果**

- 不需使用瀏覽器返回功能即可重新開始搜尋。
- 返回首頁後，旅客能再次看到繁體中文首頁表單。

## 5. 驗證搜尋結果頁不同狀態

1. 以有結果的搜尋請求開啟搜尋結果頁。
2. 以無結果或尚未有結果資料的搜尋請求開啟搜尋結果頁。
3. 以不存在的搜尋請求網址或可重現的錯誤提示情境驗證導回首頁後的中文提示與返回流程。

**預期結果**

- 有結果、無結果與錯誤提示情境下，返回首頁入口皆可見且可操作。
- 搜尋結果頁標題、排序、推薦與空狀態文案均為繁體中文。
- 若發生找不到搜尋請求的情境，首頁 alert 也應為繁體中文。

## 6. 建議自動化測試

執行以下命令驗證主要迴歸風險：

```powershell
ruby bin/rails test
ruby bin/rails test:system
```

若只想先驗證本功能關聯流程，至少應覆蓋：

- 搜尋送出後的成功提示與狀態頁中文文案
- 搜尋結果頁中文標題與返回首頁入口
- 找不到搜尋請求時導回首頁的中文提示
