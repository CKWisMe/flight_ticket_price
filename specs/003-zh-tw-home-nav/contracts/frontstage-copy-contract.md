# Frontstage Copy Contract

## 目的

定義目前旅客可見前台頁面的最小文案與導覽契約，讓 `zh-TW` 語系切換與搜尋結果頁返回首頁入口有一致的驗收基準。

## 適用頁面

| `page_key` | 頁面 | 必備文案群組 | 必備操作 |
|-----------|------|--------------|----------|
| `search_requests.new` | 首頁 | 主標題、欄位標籤、送出按鈕、驗證提示 | 建立搜尋請求 |
| `search_requests.show` | 搜尋狀態頁 | 狀態標題、摘要欄位、請求狀態標示、來源狀態說明、查看結果按鈕 | 查看搜尋結果 |
| `search_results.show` | 搜尋結果頁 | 結果標題、排序文案、推薦區、狀態標示、空狀態提示、返回首頁按鈕 | 查看結果、回到首頁、開啟訂票連結 |

## 語系契約

- 所有適用頁面的核心導覽與提示文案預設使用 `zh-TW`。
- 英文可保留於必要專有名詞、來源代號、幣別代碼或外部供應商名稱，但不得成為唯一的核心操作文案。
- flash、空狀態、驗證提示與按鈕名稱都屬於本契約範圍。

## 搜尋結果頁返回首頁契約

| 欄位 | 契約 |
|------|------|
| `action_key` | `search_results.back_to_home` |
| `target` | `root_path` |
| `label` | 明確表示返回首頁或重新開始搜尋的繁體中文文案 |
| `visibility` | `with_results`、`empty_results`、`error_notice` 三種狀態皆可見 |
| `accessibility` | 可使用鍵盤啟用，名稱可被輔助工具辨識 |

## 驗收條件

1. 首頁、搜尋狀態頁、搜尋結果頁不再出現僅英文的核心導覽按鈕或頁面標題。
2. 搜尋流程中的成功與失敗提示以繁體中文顯示。
3. 搜尋結果頁在所有狀態下都提供返回首頁入口，且可直接導向首頁。
4. 搜尋狀態頁與搜尋結果頁的狀態標示以繁體中文顯示。
5. 現有旅客主流程測試可用繁體中文斷言通過。
